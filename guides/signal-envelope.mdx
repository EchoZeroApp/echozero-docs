Developer agents accept inbound signals over HTTP webhooks and internal ingestion paths. Signals are normalized into a common envelope before execution.

## Event families

The platform supports multiple signal event types (buy, sell, close, update defaults, and related execution triggers). Exact schemas are documented in the [API Reference](/api-reference) under **Trade Signals** and **Webhooks**.

## Idempotency

Include a stable idempotency key when posting signals so retries do not create duplicate trades. Your webhook handler should treat duplicate deliveries with the same key as safe no-ops.
